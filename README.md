# Inventory Assist

**Course:** CPSC 471 — Database Management Systems  
**Group:** G-3  
**Members:** Geffery Addai, Emmanuel Buhari, Kevin Igelka, CJ Obi  

An inventory tracking web application for small to medium businesses. Supports two roles — Admin and Stock Clerk — with full product, supplier, warehouse, purchase order, and transaction management backed by a MySQL database.

---

## Tech Stack

| Layer    | Technology                          |
|----------|-------------------------------------|
| Frontend | React 18 + Vite + Tailwind CSS      |
| Backend  | Node.js + Express                   |
| Database | MySQL 8                             |

## Local Setup

### Prerequisites

- Node.js 18+
- MySQL 8 running locally (or a remote MySQL instance)

---

### Step 1 — Database

1. Open your MySQL client and create the database:
   ```sql
   CREATE DATABASE IF NOT EXISTS InventoryAssist;
   ```

2. Run the schema to create all tables, indexes, and triggers:
   ```bash
   mysql -u root -p InventoryAssist < server/schema.sql
   ```

3. Seed demo data:
   ```bash
   cd server
   npm install
   node seed.js
   ```

---

### Step 2 — Backend

1. Copy the environment template and fill in your values:
   ```bash
   cp server/.env.example server/.env
   ```

2. Edit `server/.env` with your MySQL credentials and a JWT secret (see `.env.example` for all required fields).

3. Start the backend:
   ```bash
   cd server
   npm run dev
   ```
   The API will be available at `http://localhost:3001`.

---

### Step 3 — Frontend

```bash
cd my-react-app
npm install
npm run dev
```

Open `http://localhost:5173` in your browser.

## Features

**Admin**
- Dashboard with KPI cards, low-stock alerts, and recent transactions
- Product management with variant support and bulk CSV/XLSX import
- Category, supplier, and warehouse management with detail views
- Purchase order creation (3-step wizard) with status progression
- Reorder suggestions (auto-generated by DB trigger) — convert or dismiss
- Reports: Inventory Valuation, Sales Trends, Supplier Performance
- User management with role-specific fields

**Stock Clerk**
- Dashboard scoped to assigned warehouse
- Read-only inventory view with stock status indicators
- Record transactions (Sale, Receipt, Adjustment, Return) with live stock preview
- Personal transaction history with filters

---

## Database Design

- **12 entity types** — User, Admin, StockClerk, Product, ProductVariant, Category, Supplier, Warehouse, PurchaseOrder, OrderItem, InventoryTransaction, ReorderSuggestion
- **1 weak entity** — OrderItem (depends on PurchaseOrder)
- **15 relationships** including 3 M:N junction tables (Product_Supplier, StoredIn, Admin_Warehouse)
- **3 triggers** — auto-decrement stock on Sale, auto-increment on Receipt/Return, auto-update on PO delivery
- **1 scheduled event** — auto-dismiss stale reorder suggestions after 30 days
- **13 indexes** on foreign keys and common filter columns
